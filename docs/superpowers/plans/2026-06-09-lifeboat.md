# Lifeboat Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship the Lifeboat shutdown watcher (GitHub Actions cron + alerting), the public directory site (Astro, Cloudflare Pages-ready), the strike-kit generator, and the first real rescue content — production-ready, zero fake data.

**Architecture:** Zero-dependency Python watcher (stdlib only, unittest) polls HN/Reddit/Google News, dedups via committed JSON state, scores shutdown-language + engagement, and emits alert payloads that a GitHub Actions workflow turns into Issues. Astro static site renders a directory of curated rescue guides (content collection) plus an automated signals feed. A small Python generator scaffolds new rescue pages from event records.

**Tech Stack:** Python 3.14 stdlib (urllib, xml.etree, unittest), GitHub Actions + `gh` CLI, Astro 5 (Node 24), Cloudflare Pages (static output), no paid APIs.

### Task 3: Scoring (TDD)

**Files:** Create `watcher/lifeboat_watcher/scoring.py`, `watcher/tests/test_scoring.py`

- [ ] Tests with REAL headline strings:
  - true-positives score ≥ ALERT_THRESHOLD: e.g. "X is shutting down on July 31", "Sunsetting X after 8 years", "X to discontinue service".
  - true-negatives score < threshold: layoffs news, "Shutting down my side project" (personal blog, low engagement), game-server closures, "Ask HN: why do startups shut down?".
  - engagement boosts score (same title, 500 HN points > 5 points).
- [ ] Implement: tiered regex confidence (strong: "is shutting down|will shut down|sunsetting|end of life|discontinuing|winding down" in title; weak: in body), negative patterns (layoff|laid off|game server|minecraft|"ask hn"…), engagement multiplier, ALERT_THRESHOLD constant.
- [ ] Run tests → PASS. Commit `feat: shutdown-language scoring`.

### Task 4: Orchestrator CLI + Actions workflow

**Files:** Create `watcher/lifeboat_watcher/run.py`, `.github/workflows/watch.yml`, `watcher/state/seen.json`

- [ ] `python3 -m lifeboat_watcher.run --out alerts/` : polls all queries×sources (per-source try/except, log to stderr), dedup, score, writes one `alerts/<slug>.json` per above-threshold event (title/url/score breakdown/links/issue markdown body) and updates state. - [ ] Workflow: cron `*/30 * * * *` + manual dispatch; checkout, run watcher, `git commit` state if changed, loop alerts: `gh issue create --title "🛟 <title>" --body-file …` with label `shutdown-alert` (create label step idempotent). Concurrency group to prevent overlap.
- [ ] Local smoke test: run CLI against live network, confirm alerts/state written, valid JSON. Commit `feat: watcher CLI and Actions cron`.
